# Seller AI Assistant — Личный кабинет продавца

Веб-приложение для управления объявлениями с интегрированным AI-ассистентом. Помогает продавцам оптимизировать описания товаров и оценивать их рыночную стоимость с помощью LLM (Llama 3 via Groq).

## 🚀 Основные возможности
- **AI-копирайтер**: Улучшение описания товара с учетом категории и характеристик.
- **AI-оценщик**: Анализ рыночной стоимости на основе контекста объявления.
- **Умные черновики**: Автоматическое сохранение изменений в `localStorage` через **Zustand**.
- **Динамические формы**: Поля характеристик адаптируются под выбранную категорию (Авто, Недвижимость, Электроника).
- **Валидация**: Индикация объявлений, требующих доработки (на основе пустых обязательных полей).

## 🛠 Технологический стек
- **Frontend**: React 18, TypeScript, Vite.
- **UI-библиотека**: Ant Design (Antd).
- **State Management**: Zustand (Client state & Persist), React Query (Server state).
- **Network**: Axios.
- **AI Integration**: Groq Cloud API (Model: Llama 3.3 70B).

## 📦 Запуск проекта

### 1. Требования
- Node.js v20+
- Ключ API от [Groq Console](https://console.groq.com/)

### 2. Настройка окружения
Создайте файл `.env` в корневом каталоге и добавьте ваш ключ:
```env
VITE_OPENAI_API_KEY=ваш_ключ_gsk_...
```
## 💡 Приятые технические решения

### 1. Интеллектуальные AI-функции
- **Умный парсинг цены**: Реализован алгоритм `parseAiPrice`, который извлекает среднее арифметическое из сложных текстовых диапазонов (например, "115 000 – 135 000 ₽"), позволяя подставить число в инпут одним кликом.
- **Динамические Popover-состояния**: Для функций "Улучшить описание" и "Узнать цену" создана единая система состояний (`idle`, `loading`, `success`, `error`). При ошибках интерфейс визуально меняется без использования внешних CSS-библиотек (через `overlayInnerStyle`).
- **Prompt Engineering**: Настроены специализированные инструкции для AI, чтобы получать ответы в строгом формате, удобном и для чтения человеком, и для программного парсинга.

### 2. Работа с черновиками (Persistence)
- **Автосохранение**: Любое изменение в форме автоматически попадает в `localStorage` через Zustand.
- **Smart Restore**: При загрузке страницы система сопоставляет данные с сервера и локальный черновик. Если найдены отличия, пользователю предлагается восстановить работу.

### 3. Исправления и доработки Бэкенда
В ходе разработки были внесены критические правки на стороне сервера
- **DTO Fix**: Добавлен возврат `id` в моделях объявлений (изначально API не отдавал идентификатор, что блокировало роутинг).
- **CORS & Proxy**: Проблема кросс-доменных запросов на этапе разработки решена через настройку Vite Proxy, что позволило избежать конфликтов портов.
- **Server-side Sorting**: Реализована логика сортировки по цене на стороне БД (через LINQ), так как базовая версия API не поддерживала упорядочивание контента.
